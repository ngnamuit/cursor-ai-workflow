# Phase 2 – Docs (Research → PRD Bridge)
### Tại sao phải có “Research → PRD Bridge”?
```commandline
Vì:
Research = suy nghĩ rời rạc
Code = hành động cụ thể
PRD = cầu nối logic
```
=> Cursor rất mạnh khi có PRD rõ.

### PRD tối thiểu nên có
```commandline
1. Executive Summary (3 câu)
2. Background & Context
3. User Personas + JTBD
4. Requirements (MoSCoW)
5. Success Metrics
6. Scope / Non-Goals
7. Open Questions
```

## Cách dùng Cursor cho phase này
```commandline
Here is my research.md.
Help me convert this into a clear PRD.

Rules:
- No implementation details
- Be explicit about assumptions
- Call out open questions clearly


Sau đó:
- Lưu thành PRD.md
- Đây sẽ là nguồn chân lý (source of truth)
```
